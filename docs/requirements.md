# 📑 [기획 명세서] 내 성격에 딱 맞는 IT 부캐 찾기 (MBTI 테스트)

## 1. 프로젝트 개요 (Overview)

- **프로젝트명**: 내 성격에 딱 맞는 IT 부캐 찾기 테스트 (My IT Alter-Ego MBTI)
- **배경 및 문제 정의**:
    - IT 업계(기획, 개발, 디자인 등)로 진입하려는 취준생이나 주니어들이 자신의 성향이 실무에서 어떤 스타일로 발현되는지 잘 알지 못함.
    - 기존의 딱딱한 적성 검사 대신, 대중적인 MBTI 포맷을 차용하여 쉽고 트렌디하게 자신의 업무 성향을 진단할 수 있는 서비스가 필요함.
- **목표**: 8개의 질문을 통해 사용자의 MBTI 차원(E/I, S/N, T/F, J/P)을 분석하고, 그에 맞는 IT 직장인 페르소나를 매칭함. 실시간 누적 참여자 수를 보여주어 소셜 공유(바이럴)를 유도함.

---

## 2. 핵심 기능 요구사항 (Requirements)

- **사용자 플로우**: 메인 화면 ➔ 테스트 시작 ➔ 8개 성향 판별 질문 답변 (각 차원당 2문항) ➔ 결과 분석 로딩 ➔ 최종 결과 화면 (16가지 조합 중 1개 렌더링) ➔ 공유하기
- **백엔드(Supabase) 연동**:
    - **참여 시 데이터 적재**: 유저가 테스트를 마치면 최종 MBTI 결과 데이터(예: 'INTJ', 'ENFP')와 선택한 답변 리스트가 DB에 저장되어야 함.
    - **실시간 카운터**: 메인 화면 진입 시, 현재까지 저장된 DB의 전체 Row 수를 읽어와 `현재까지 총 N명이 부캐를 찾았습니다` 배너를 실시간으로 노출해야 함.

## 3. UI/UX 와이어프레임 및 디자인 가이드

### 3-1. 메인 화면 (Home)

- **타이틀**: "출근길 내 모습으로 알아보는 'IT 부캐' 테스트"
- **서브 타이틀**: "협업 스타일부터 위기 대처법까지, 내 MBTI 유형에 맞는 IT 직무 페르소나는?"
- **실시간 카운팅 배너**: `👥 이미 N명의 동료들이 자신의 부캐를 확인했어요!` (눈에 띄는 백그라운드 컬러 적용)
- **CTA 버튼**: `[내 IT 부캐 확인하러 가기 ➔]` (애니메이션 효과 포함)

### 3-2. 질문 화면 (Quiz)

- 상단에 `[ 1 / 8 ]` 형태의 현재 진행 상황과 시각적인 프로그레스 바(Progress Bar) 노출.
- 질문 텍스트는 중앙에 크게 배치, 하단에 2개의 선택지 버튼을 세로로 배치. 버튼 클릭 시 부드러운 페이드아웃 효과와 함께 다음 질문으로 이동.

### 3-3. 결과 화면 (Result)

- **상단**: 나의 IT 부캐 타이틀 (예: "철두철미한 코드 마스터", "아이디어 폭발 기획자") 및 MBTI 4자리 알파벳 노출.
- **본문**:
    - 해당 성향의 일상 특징 3가지 요약 (Bullet Point)
    - 최고의 협업 파트너 (찰떡궁합 유형) / 최악의 협업 파트너 환상의 케미 안내.
- **하단**: `[테스트 다시 하기]` 및 `[결과 링크 복사하기]` 공유 기능 버튼.

## 4. 성향 판별 로직 및 질문 세트 (MBTI 4개 차원)

> 💡 **알고리즘 가이드**: 질문은 총 8개이며, 각 성향 축(E/I, S/N, T/F, J/P)별로 2문항씩 배정됩니다. 선택지에 따라 해당 알파벳 점수가 1점씩 누적되며, 최종적으로 점수가 더 높은 알파벳 조합이 유저의 MBTI가 됩니다. (동점일 경우 I, N, T, P를 우선순위로 둡니다.)
> 

### [E vs I 차원: 에너지의 방향]

#### Q1. 새로운 대규모 IT 컨퍼런스나 네트워킹 파티에 참여했을 때 나의 행동은?

- **[선택지 A]** 처음 보는 사람들에게도 적극적으로 명함을 건네며 링크드인 일촌을 맺는다. ➔ **(E +1)**
- **[선택지 B]** 아는 사람 몇 명과 주로 대화하거나, 조용히 세션을 들은 후 집으로 퇴근한다. ➔ **(I +1)**

#### Q2. 대면 미팅과 슬랙(Slack)/카카오톡 업무 메세지 중 내가 더 선호하는 소통 방식은?

- **[선택지 A]** 말로 빠르게 치고받으며 회의실에서 화이트보드에 적으며 얘기하는 게 편하다. ➔ **(E +1)**
- **[선택지 B]** 생각과 텍스트를 정리해서 비동기식 메세지로 소통하는 게 훨씬 편하다. ➔ **(I +1)**

### [S vs N 차원: 정보 인식의 방식]

#### Q3. 새로운 서비스 기획 아이디어 회의를 할 때, 내가 더 집중하는 부분은?

- **[선택지 A]** 경쟁사 성공 사례, 현재 가용 리소스, 기존 데이터 지표 등 '현실적인 실현 가능성' ➔ **(S +1)**
- **[선택지 B]** "이 기능이 세상을 어떻게 바꿀까?" 같은 '새롭고 실험적인 비전과 확장성' ➔ **(N +1)**

#### Q4. 선배가 "요즘 인공지능(AI) 트렌드가 급변하고 있네요"라고 말했을 때 내 머릿속 떠오르는 생각은?

- **[선택지 A]** "맞아요, 당장 내년 우리 제품에 적용할 만한 LLM API 단가부터 알아봐야겠어요." ➔ **(S +1)**
- **[선택지 B]** "조만간 AI가 인간의 UI를 완전히 대체하는 영화 같은 세상이 오지 않을까요?" ➔ **(N +1)**

### [T vs F 차원: 판단과 결정의 방식]

#### Q5. 동료가 "제가 밤새서 기획서 써왔는데, 에러가 너무 많이 나요..."라며 괴로워할 때 나의 첫마디는?

- **[선택지 A]** "어디 봐봐요. 콘솔 로그에 어떤 에러 코드가 찍히는지 확인해 드릴게요." ➔ **(T +1)**
- **[선택지 B]** "어떡해요, 밤새 고생하셨을 텐데 진짜 속상하시겠어요. 커피 한잔 마시고 해요." ➔ **(F +1)**

#### Q6. 프로젝트 피드백 세션에서 우리 팀 제품의 치명적인 단점을 지적받았을 때 내가 느끼는 감정은?

- **[선택지 A]** 제품을 개선할 수 있는 객관적인 팩트 피드백이므로 유익하다고 생각한다. ➔ **(T +1)**
- **[선택지 B]** 우리 팀이 열심히 고생해서 만든 건데, 노력을 몰라주는 것 같아 조금 서운하다. ➔ **(F +1)**

### [J vs P 차원: 이행 및 생활 양식]

#### Q7. 내일 아침까지 끝내야 하는 중요한 스프린트 태스크(Task)가 주어졌을 때 내 스타일은?

- **[선택지 A]** 오늘 시간대별로 작업 계획을 쪼개서 일정을 완벽히 통제하며 끝낸다. ➔ **(J +1)**
- **[선택지 B]** 일단 다른 급한 일부터 하다가, 마감 직전 초인적인 몰입력을 발휘해 끝낸다. ➔ **(P +1)**

#### Q8. 프로젝트 도중 갑작스러운 클라이언트의 요구사항 변경으로 기존 기획을 대폭 수정해야 한다면?

- **[선택지 A]** 철저하게 짜둔 일정과 WBS(일정표)가 꼬여서 엄청난 스트레스를 받는다. ➔ **(J +1)**
- **[선택지 B]** "IT 하다 보면 그럴 수 있지!"라며 유연하게 생각하고 임기응변으로 대처한다. ➔ **(P +1)**
